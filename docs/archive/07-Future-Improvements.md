# 07 - التحسينات المستقبلية (Future Improvements)

يوضح هذا المستند المناطق التي قد تحتاج إلى تطوير أو إعادة هيكلة في المستقبل مع نمو النظام.

## 1. الأداء والتحجيم (Performance & Scalability)
### تقسيم الصفحات (Pagination)
- **الحالة الحالية**: يتم تحميل جميع السجلات في صفحة واحدة (`fetchAll`).
- **المشكلة**: مع تجاوز عدد السجلات 5,000 - 10,000، ستصبح الواجهة بطيئة جداً.
- **الحل المقترح**: تطبيق `Pagination` (عرض 50 سجل في كل صفحة) في الـ API والواجهة الأمامية.

### تحسين قاعدة البيانات
- **الحالة الحالية**: استخدام `SQLite`.
- **الحدود**: ممتازة للاستخدام المحلي ولحجم بيانات متوسط.
- **التحسين المستقبلي**: في حال تعدد المستخدمين (Multi-user) أو الوصول المتزامن الكثيف، يفضل الانتقال إلى `MySQL` أو `PostgreSQL`.

## 2. إدارة الملفات (Storage)
### تنظيف الملفات القديمة
- **الحالة الحالية**: الاحتفاظ بجميع ملفات Excel المرفوعة للأبد.
- **المشكلة**: استهلاك مساحة القرص.
- **الحل المقترح**: إضافة وظيفة مجدولة (Cron Job) لحذف الملفات الأقدم من سنة.

## 3. تجربة المستخدم (UX)
### تقليل الواجهات (Inline Management)
- **الهدف**: تجنب إنشاء صفحات إدارة منفصلة (CRUD) لتقليل التعقيد.
- **الحل المستقبلي (Smart Inline Editing)**:
  - تمكين تعديل اسم المورد الرسمي أو البنك مباشرة من داخل بطاقة السجل (Record Card).
  - عند تعديل اسم مورد، يسأل النظام بذكاء: "هل تريد تعميد هذا الاسم لجميع السجلات السابقة؟" أو "هل تريد اعتباره اسماً بديلاً؟".
  - دمج وظائف "الإضافة" و "التصحيح" في نفس سياق العمل (Context) دون مغادرة الصفحة.

### التحرير المباشر الشامل (Seamless Table Experience)
- **المرحلة المتقدمة (Ultimate Vision)**:
  - التخلص نهائياً من "بطاقات السجل" (Popups/Modals) وصفحات الإدارة.
  - تحويل جدول السجلات نفسه إلى واجهة تفاعلية ذكية (Excel-like Experience).
  - يمكن للمستخدم النقر على أي اسم مورد أو بنك وتعديله فوراً، ليقوم النظام بتحديث القاموس، التاريخ، والسجل الحالي في لحظة واحدة.
  - الهدف: سرعة العمل القصوى وتقليل النقر (Minimum Clicks).

## 4. الأمان (Security)
### نظام الدخول (Authentication)
- **الحالة الحالية**: النظام مفتوح (مخصص للشبكة الداخلية الآمنة).
- **التحسين المستقبلي**: إضافة نظام تسجيل دخول (Login) وصلاحيات (Admin/User) إذا تقرر نشر النظام على شبكة أوسع.

## 5. الحزم المستقلة (Desktop Packaging)
- **الهدف**: تشغيل النظام كملف EXE مستقل (Portable) دون تثبيت سيرفر.
- **الحالة الحالية**: ✅ **متوافق تماماً**.
  - الاعتماد على `SQLite` (ملف محلي).
  - المسارات (Paths) تستخدم `__DIR__` (نسبية).
  - لا توجد اعتماديات خارجية (External Dependencies).
- **التنفيذ المستقبلي**:
  - استخدام أداة مثل [PHP Desktop](https://github.com/cztomczak/phpdesktop) لدمج ملفات المشروع مع متصفح Chrome وسيرفر PHP في ملف واحد.
