# سياسة الترقيم
اتبع SemVer مبسطًا للإصدارات الداخلية.

- `MAJOR.MINOR.PATCH` (مثال: `1.2.0`).  
- زيادة **MAJOR** عند تغييرات بنيوية أو كسر توافق بيانات.  
- زيادة **MINOR** للميزات الجديدة المتوافقة.  
- زيادة **PATCH** للإصلاحات الصغيرة.  
- سجل رقم إصدار القاعدة في `schema_meta` إن تغيرت بنية قاعدة البيانات.
