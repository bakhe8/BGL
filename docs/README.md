# BGL Docs — نقطة البداية
حزمة التعليمات المنظَّمة لمشروع BGL (PHP Desktop + SQLite).  
ابدأ من قسم **00-Overview** ثم انتقل إلى الإعداد وعمليات التطوير والمطابقة.

## خريطة المحتوى
- **00-Overview** — الملخص، الميزات، القاموس.
- **01-Setup** — خطوات التشغيل محليًا (PHP Desktop، Tailwind، البيئة).
- **02-User-Guide** — كيفية استخدام النظام (استيراد، مراجعة، قواميس، تصدير).
- **03-Developer-Guide** — معمارية المشروع، تدفق الباك/الفرونت، معايير الترميز.
- **04-Database** — مخطط SQLite، المعايير، والمخطط الكامل.
- **05-Matching-Engine** — التطبيع، المطابقة، البدائل، الثقة، التكامل مع شاشة المراجعة.
- **06-Excel-Processing** — قواعد قراءة ملفات Excel، التحقق، ومعالجة التواريخ/الأرقام.
- **07-UI-Design** — قواعد التخطيط، المكونات، الثيم، وملخص فئات Tailwind.
- **08-Testing** — سيناريوهات يدوية، وحدة، تكامل، وقائمة فحص الجودة.
- **09-Release-Management** — النسخ، التوزيع، النسخة المحمولة، والتغييرات.
- **10-Appendix** — حدود معروفة، خريطة طريق، أكواد الخطأ، ونقاط API الشبيهة.

## طريقة الاستخدام السريعة
1) اقرأ **00-Overview/project-summary.md** و **feature-list.md** لتفهم نطاق المنتج.  
2) طبّق خطوات **01-Setup/install-project.md** وشغّل نسخة العمل.  
3) استخدم **02-User-Guide** أثناء التجربة الأولى.  
4) للمطورين: اطلع على **03-Developer-Guide/project-architecture.md** و **04-Database/schema.sql**.  
5) عند تعديل المنطق: راجع **05-Matching-Engine** و **06-Excel-Processing**.  
6) قبل أي إصدار: مرّ على **08-Testing/QA-checklist.md** ثم **09-Release-Management**.
