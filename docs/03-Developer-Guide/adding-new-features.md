# إضافة ميزات جديدة
مسار آمن لإضافة ميزة دون كسر الاستقرار.

1) **تعريف المستخدم والقيمة**: اكتب User Story قصيرة وما الذي يجب أن يراه المستخدم.  
2) **تحديث البيانات**: هل نحتاج جدول/حقل جديد؟  
   - أضف migration في `database/migrations/` + حدث `docs/04-Database/schema.sql`.  
3) **الخدمات**: أضف/عدل Service أو دوال جديدة واضحة، مع اختبارات وحدة للمنطق.  
4) **الواجهة**: عدّل شاشة واحدة في كل مرة، استخدم API layer بدلاً من طلبات موزعة.  
5) **التوثيق**: حدّث ملفات الدليل ذات الصلة (User-Guide/Developer-Guide).  
6) **الاختبارات**: مرّ على `08-Testing/QA-checklist.md` وأضف سيناريوهات تكامل عند الحاجة.

مؤشر نجاح: الميزة تعمل محليًا، الاختبارات الأساسية تمر، ولا يوجد تغيير غير مبرر في الجداول.
