# Coding Standards
معايير سريعة للباك‑إند والفرونت.

## PHP
- التزم بـ PSR-12 للتنسيق والمسافات.  
- استخدم الأنواع الصارمة `declare(strict_types=1);` في كل ملف.  
- صنّف الكود إلى Controllers/Services/Repositories، بدون منطق عمل في الـ Controllers.  
- استخدم حقن التبعيات (تمرير Repositories/Helpers للـ Services) بدل الاستدعاء الساكن.  
- الدوال قصيرة وواضحة؛ ملف الخدمة لا يتجاوز 200 سطر إن أمكن.

## JavaScript
- ES modules قدر الإمكان، بدون متغيرات عالمية عشوائية.  
- افصل منطق API (ملف api.js) عن العرض (render.js) والحالة (state.js).  
- تعامل مع الأخطاء بإرجاع رسائل موحدة للواجهة.

## CSS / Tailwind
- اعتمد فئات مسماة واضحة؛ تجنب تكرار تنسيقات مخصصة خارج قائمة الفئات الموثقة.  
- استخدم CSS variables للألوان والخطوط المحددة في دليل الثيم.

## توثيق
- التعليقات للمنطق غير البديهي فقط.  
- أي تغيير في قاعدة البيانات يجب أن يوثق في `docs/04-Database/migrations.md` و `schema.sql` عند التحديث.
