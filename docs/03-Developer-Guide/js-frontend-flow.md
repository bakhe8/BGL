# JS Frontend Flow
الواجهة بسيطة (بدون إطار). هذا تدفق العمل القياسي.

1) **State**: كائن مركزي يحتفظ بالسجلات الحالية، حالة التحميل، وأي فلاتر.  
2) **API Layer** (`api.js`): دوال `fetch` موحدة ترجع `{ success, data, error }`.  
3) **Actions**: وظائف تغلف تدفقات كاملة (استيراد ملف، حفظ قرار، إنشاء مورد).  
4) **Render**: تحديث DOM بناءً على الحالة (جداول، بطاقات، إشعارات).  
5) **Events**: استمع للأزرار/النماذج، استدع actions، ثم أعد render.

## قواعد
- استخدم `async/await` مع try/catch، واعرض رسالة خطأ موحدة أعلى الشاشة.  
- افصل تنسيق الأرقام/التواريخ في ملف helper واحد.  
- لا تحفظ الحالة في DOM attributes؛ استخدم state واحد لتفادي التشتت.  
- اجعل كل شاشة لها ملف تهيئة مستقل (مثال: `review-screen.js`, `dictionary-screen.js`).
